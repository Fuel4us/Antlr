Introdução

Este projeto foi realizado no âmbito da disciplina de Linguagens e Programação do 2º ano da licenciatura de Engenharia Informática do ISEP. Neste relatório pretendemos demonstrar os conhecimentos adquiridos ao longo do semestre.

Vamos usar os nossos conhecimentos para desenvolver uma solução sobre o tema que nos foi proposto: Gerir Salas do ISEP. Alguns dos problemas que nos surgiram foram:

  • Manipulação de XML
  • Usar a sintaxe correta para ligar XML e Schemas
  • Criação de 30 salas sem as tornar muito iguais
  • Conseguir por uma transformação usando o sort a funcionar
  • Perceber bem a hierarquia que se pode usar no relatorio

Análise

- Especificação do problema:

Na resolução deste trabalho o importante era manipular bem as propriedades das salas do ISEP que por são ainda constituidas por outros atributos. No nosso trabalho usamos todas os atributos dados no enunciado.

  • Sala
  • Local
  • Área
  • Mobilia
  • Equipamento Informático (Hardware)
  • Equipamento elétrico
  • Calendário da limpeza das salas
  • Calendário da manutenção
  • Avarias presentes nas salas

Linguagem

- Criação de um XML com as especificações de um XML Schema:

Inicialmente iamos começar pela criação do XML mas apercebemo-nos que a maneira mais correta seria primeiro definir o schema com as regras do nosso problema e após isso passar para a resolução do XML para ser usado como base para o projeto, para efeito de teste de transformações.

No schema estão definidas todas as regras de contrução do nosso negócio de gestão de salas do isep. Algumas das regras é por exemplo alguns dos atributos como avarias ou mobilia existir mais que 1 atributo, logo dividimos essas propriedades como listas. Por exemplo o Local e a área sendo atributos que não se iriam repetir não sentimos necessidade de fazer isso.

Transformações

- Transformações:

Após a realização do Schema e do XML, partimos para as transformações.

Realizamos 7 transformações:

  • 1º Transformação para HTML: Mostra toda a informação do xml
  • 2º Transformação para HTML: Lista todas as avarias por sala
  • 3º Transformação para HTML: Mostra os calendários de limpeza e manutenção ordenados por data
  • 4º Transformação para TXT: Mostra toda a informação num ficheiro txt
  • 5º Transformação para TXT: Mostra a lista de equipamentos por sala
  • 6º Transformação para XML: Mostra o número e a área das salas por ordem descendente de área
  • 7º Transformação para XML: Mostra a lista de equipamentos por quantidade superior a 10

Conclusão

Após bastantes horas investidas neste projeto sentimos que os nossos conhecimentos a XML, XSL, XSD evoluíram bastante!

Este trabalho foi nos pedido no inicio do mês de Maio o que nos deu bastante margem de manobra. Mas claro nenhum trabalho é realmente perfeito. Sentimos algumas dificuldades na realização das transformações e podiamos ter nos organizado melhor em termos de tempo dispendido no trabalho, mas apesar das adversidades realizamos um trabalho bom e as transformações estão todas funcionais.
